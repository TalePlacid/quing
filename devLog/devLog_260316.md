### 작업 1. 경량 테스트 구조 설계

#### 1. 문제
- QtCreator 내부 콘솔에서 한글 입력이 깨지는 문제가 발생.
#### 2. 원인 분석
- 컴파일러 인코딩은 UTF-8인데, QtCreator 내부 콘솔은 CP949인코딩이라 한글 깨짐 발생.
- 컴파일러 인코딩을 CP949로 내리는 방식은 차후 확장성에 장애가 될 가능성.
- QtCreator 내부 콘솔 인코딩을 변경하는 방법은 찾지 못함.
- QString으로 문자열 인코딩을 강제할 수는 있으나, 도메인 테스트가 Qt 라이브러리에 의존하게 됨.
#### 3. 해결방안
- QString으로 문자열 인코딩을 강제하되, 유틸리티 클래스로 분리.
- 유틸리티 클래스에 std::string으로 입력하고, 내부에서 QString을 활용해 문자열을 강제 인코딩한 뒤, 다시 std::string으로 반환.
#### 4. 작업 내용
- Local8BitStringConverter 추가
#### 5. 결과
- 자체적인 경량 테스트 구조 설계.
