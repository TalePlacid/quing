### 작업 1. UUID 정책 설정

#### 1. 문제
- Task 엔티티의 속성 UUID를 추가하기 위해 UUID 생성기를 만들어야한다는 생각이 듬.
- `<random>`라이브러리를 활용해 UUID 생성기를 구현.
- 하지만 해당 UUID는 1초 내로 다수의 UUID가 생성되면 충돌 위험이 있음.
- 현재 시스템상 1초내에 다수의 UUID가 생성될 일은 없음.
- 하지만 크로스 플랫폼 확장을 고려했을 떄, 앞으로도 그럴 거라고 확신할 수 없다.
- 안정적으로 UUID를 생성해주는 QT 라이브러리나, Windows OS API는 종속성 문제 때문에 쓸 수 없다.
#### 2. 원인 분석
- UUID를 도메인 레벨에서 생성해야한다는 것부터가 논리 오류.
- UUID는 그저 Task 엔티티를 구별하는 고유 문자열에 불과함.
- 따라서 Task 엔티티에서 정해야할 것은 정책뿐. 
#### 3. 해결방안
- 현재 시점에서는 정책만 정하고, 생성기는 UI 설계때 정의한다.
#### 4. 작업 내용
- 구현 편의성을 위해 QT 라이브러리에서 반환하는 UUID 형식
  (XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX)
  을 기본 정책으로 삼음.
- 생성기 자체는 이후 바운더리 영역에서 정의.
- 차후 확장시에는 정책에 맞게 포맷팅하여 주입.
#### 5. 결과
- Task의 UUID 정책 설정.
