### 작업 1. 테스트 분류 정책 결정

#### 1. 문제
- 각 테스트에 대해 분류를 어떻게 하고 어떤 디렉토리 구조로 관리할 지 정해져있지 않음.
#### 2. 고민
- Service : 어플리케이션 계층. test\application 예하로.
- ActionHandler : 사용자 입력을 중계하는 바운더리 계층. test\boundary 예하로.
- ViewModel : UI 상태값 관리. 프레젠테이션 계층이라고 볼 수 있음.
- Qml : 실제 표현(그리기) 계층. 마찬가지로 프레젠테이션 계층이라고 볼 수 있음.
- ViewModel과 Qml은 개념적으로 같은 계층이지만 기술적으로는 다름. 아예 테스트 기법적으로 다름.
- 테스트 기법이 다른 Qml은 아예 별도로 분리하는 게 좋겠음. 
#### 3. 해결방안
- test\cpp, test\qml로 디렉토리 분리.
- test\cpp\에는 domain, application, boundary, presentation으로 나눔.
- test\qml\에는 qml용 경량 텍스트 구조와 테스트 파일들을 구성.
#### 4. 작업 내용
- 디렉토리 구조 조정.
- 테스트 분류 정책 결정
#### 5. 결과
- application, boundary, presentation 테스트 추가 필요.
- Qml 경량 테스트 구조 설계 필요.
